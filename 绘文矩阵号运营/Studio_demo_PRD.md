## 一、模块概述

### 背景

讯飞绘文主产品面向 C 端经营型用户。矩阵号运营模块面向有批量账号运营需求的 B 端用户（MCN、品牌自运营团队等），以 SaaS 服务形式独立承接，与主产品做产品形态上的分割。

**典型用户画像（参考：懂车帝小红书业务）**
- 管理 50 个账号，每人管 7～8 台手机
- 每天每账号发布 2～3 篇内容，目标日发布量 2500 条
- 当前痛点：内容生产依赖手动操作、发布全靠手机手动、无跨账号聚合数据

### 核心定位

帮助 B 端用户管理多账号矩阵，批量生产和分发内容，并追踪互动效果。

## 三、批量任务模块

### 3.1 任务类型

| 类型 | 说明 |
|------|------|
| 内容生成型 | 批量生成内容，结果进入内容库（草稿/待审核状态） |
| 内容发布型 | 将库中已有内容分发至多账号。**流程：内容选择 -> 平台/账号配置 -> 执行。** |
| 生成＋发布型 | 批量生成内容后，由系统自动完成 AI 预审与发布，全程后台执行（Autopilot）。**流程：Skill 选择 → 全局设置（生成参数 + 审核策略）→ 数据表格（主题 × 账号）→ 后台自动执行。** |

> **差异化流程说明**：
> - **生成类任务**：遵循「Skill 选择 -> 全局设置 -> 逐行参数表」的逻辑。
> - **发布类任务 (如 Skill 41)**：遵循「内容选择 -> 批量配置 -> 执行」的逻辑，系统将根据选中的内容和目标账号自动填充默认发布配置（记忆用户习惯）。
> - **生成+发布类任务 (如小红书一键模板创作+发布)**：遵循「Skill 选择 -> 全局设置（生成参数 + 审核策略）-> 数据表格（主题 × 账号）-> 后台自动执行」的逻辑。发布配置跟随账号，不设全局默认。提交后系统全程自动运行，用户仅在 AI 标记内容或发布失败时介入。

### 3.3 任务状态

| 状态 | 触发条件 | 用户可做什么 |
|------|----------|-------------|
| 待执行 | 任务提交后排队 | — |
| 生成中 | 内容批量生成中 | 中途查看已生成内容、对已就绪／待审核内容进行审核操作 |
| 待审核 | 生成完毕 + 人工审核策略 | 进入批量审核 |
| 发布中 | 内容进入发布队列 | 查看发布日志 / 暂停 |
| 已完成 | 所有内容发布完毕 / 入库 | 查看数据 / 归档 |
| 失败 | 生成过程有内容失败 | 重试失败项 |

### 3.4 任务行展开：内容卡片字段

- 封面（配图或 Emoji 占位）
- 内容状态点（就绪 / 待审核 / 已发布 / 失败 / 生成中）
- 标题（两行截断）
- 目标账号
- 平台
- 状态文字标注

#### 内容状态说明

| 状态 | 含义 |
|------|------|
| 就绪 | AI 预审通过，可发布 |
| 待审核 | AI 预审有标记，需人工确认 |
| 已发布 | 已成功发布至平台 |
| 失败 | 生成失败 |
| 生成中 | 尚未完成生成 |

#### 按任务状态差异化展示内容

| 任务状态 | 展开区额外信息 | 可用操作 |
|---------|--------------|--------|
| 生成中 | 已生成内容实时渲染，状态为就绪或待审核 | 单篇审核（通过／编辑）、立即发布就绪项、导出打包 |
| 待审核 | 全部已生成 | 进入批量审核、导出打包 |
| 有失败项 | 顶部失败告警 + 失败原因 | 一键重试失败项 |
| 发布中 | 发布日志（内容/账号/计划时间/状态） | 查看日志、暂停发布 |
| 已完成 | 数据摘要（阅读/收藏/评论/私信均值） | 查看数据看板、归档导出 |

> **流式审核说明**：生成中状态下，内容每篇完成后立即进行 AI 预审，预审完成即标注「就绪」或「待审核」。用户不需等全部内容生成完毕，可立即对已完成的内容进行单篇审核、批量发布就绪项等操作。

#### 展开区操作按钮说明

| 按钮 | 出现时机 | 行为 |
|------|---------|------|
| 立即发布 X 篇就绪项 | 生成中，有就绪内容时 | 将所有「就绪」内容批量加入发布队列，弹出确认后执行 |
| 进入批量审核 | 待审核状态（所有内容生成完毕 + AI 预审全部跑完后出现）| 打开内容抽屉，context 限定为「待审核」内容列表，从第一篇开始，可用上下篇翻页逐一处理（通过 / 编辑 / 丢弃） |
| 导出打包 | 生成中 / 待审核 / 已完成 | 将本批次内容打包为 ZIP 下载（格式见下方说明） |
| 一键重试失败项 | 有失败内容时 | 对状态为「失败」的内容重新触发生成任务 |
| 查看数据看板 | 已完成状态 | 跳转数据看板，自动筛选至本批次数据视图 |

**导出打包文件格式（ZIP）**

以图文内容为主，ZIP 内按篇目分文件夹，附汇总清单：

```
任务名称_导出.zip
├── 汇总清单.xlsx          # 所有内容的标题、账号、平台、状态
├── 001_内容标题/
│   ├── 文案.txt
│   ├── 配图1.jpg
│   └── 配图2.jpg（如有）
├── 002_内容标题/
│   ├── 文案.txt
│   └── 配图1.jpg
└── ...
```

- 仅导出状态为「就绪」或「待审核」的内容，失败和生成中的条目跳过
- 文件夹编号按内容生成顺序排列
- 汇总清单字段：编号 / 标题 / 目标账号 / 平台 / 状态

### 3.7 Skill 详情页与试用 (Skill Detail & Trial)

用户点击「快捷发起」中的任意 Skill 卡片，首先进入该 Skill 的**详情浮层 (Modal)**：

**左侧：Skill 介绍与参数声明**
- 展示 Skill Name、描述、核心能力标签。
- 明确划分「全局设置（整任务共用）」与「逐行参数（每篇不同）」以及具体的交付物说明。

**右侧：单次试用 (Single Run Trial)**
- 提供输入表单供用户测试当前 Skill 效果。
- **全局设置区**：上传模板、选择张数、参考图源等。（如：上传模板后，会自动识别并锁定相关参数）。
- **单篇内容参数区**：填写本条试用的具体核心变量。
- 点击「开始试运行」后，流式或 Loading 展示单条生成结果。
- 结果页提供「存入内容库」与「导出」二次操作。

**对于具备发布能力的 Skill（has_publish_capability: true），试用流程分为两阶段：**

- **Phase 1 — 配置与生成**：右侧面板除全局设置区、单篇参数区外，额外显示「发布账号」选择项（选定后自动加载该账号默认发布配置）。点击「生成并预览」后流式展示生成结果。
- **Phase 2 — 预览与确认**：展示封面图组、标题、正文及 AI 预审结果，提供三个操作：
  - **重新生成**：返回 Phase 1 调整参数重试。
  - **确认发布**：将本篇内容发布至所选账号，内容入库。
  - **⚡ 批量使用 →**：同下。

**底部操作**：
- 点击「⚡ 批量使用 →」，则关闭详情弹窗，进入「新建批量任务」弹窗并自动定位至当前 Skill（Step 1 已选中，参数不预填）。

新建批量任务的配置项会根据任务类型（由 Skill 能力属性决定）产生分支逻辑：

#### A. 生成类任务 (如图文仿写、长文复刻)
强调“从无到有”的参数输入：
- **Step 1：选择 Skill**。
- **Step 2：全局设置 (Global Settings)**。任务名、审核策略、Skill 全局参数。
- **Step 3：数据表格 (Row Parameters)**。手动填表或上传 Excel。

#### B. 发布类任务 (如 Skill 41)
强调”素材到平台”的分布分流：
- **Step 1：确定内容 (Content Selection)**。
  - 从「内容库」点击进入，或在弹窗内浏览库内素材（支持多选）。
- **Step 2：配置发布与预览 (Platform & Config Generation)**。
  - **账号匹配**：系统根据内容主题或用户历史习惯，自动填充目标账号。
  - **批量调整**：在配置预览表中，统一或单独调整发布时间、话题策略、分发设置。
  - **直接执行**：确认无误后直接进入发布队列。

#### C. 生成+发布类任务 (如小红书一键模板创作+发布)
强调”从内容创作到发布上线的全自动闭环”：
- **Step 1：选择 Skill**。
- **Step 2：全局设置 (Global Settings)**。任务名、生成参数（配图模板、图源等）、审核策略（AI 自动预审 / 全人工审核）。**发布配置不在此处设置，跟随各账号默认配置。**
- **Step 3：数据表格 (Row Parameters)**。每行 = 一个主题 × 一个发布账号：
  - **账号组快速分配**：支持下拉选择账号组，系统按组内顺序自动映射至各行。
  - 同一账号可在多行重复出现，发布间隔由系统自动处理。
  - 账号选定后自动加载该账号默认发布配置；状态显示「✓ 已就绪」或「⚠ 待配置」。
  - 可点击「展开配置」抽屉查看或调整该行完整小红书发布参数。
- **后台执行 (Autopilot)**：提交后系统自动完成生成 → AI 预审 → 发布全流程，用户仅在 AI 标记内容或发布失败时介入。所有内容发布完成后入内容库，打「矩阵批次」标签。

**审核策略说明**
- **AI 自动预审**：每篇生成后立即预审，无问题 → 就绪；有问题 → 待审核。用户可批量通过就绪项，逐条处理待审核项。
- **全人工审核**：全部内容生成完毕后进入「待审核」，需逐一人工确认后发布。

---

### 3.9 任务状态流转

```
提交 → 待执行 → 生成中
                  ↓
         [AI 自动预审]           [全人工审核]
         就绪 + 待审核并存         全部待审核
         用户批量发布就绪项         用户批量审核
                  ↓ 发布
               发布中
                  ↓
               已完成

失败路径：生成中 → 部分内容失败 → 用户重试失败项
```

## 四、内容库模块

### 4.1 定位

内容库是所有内容的持久资产管理中心，与主产品草稿箱**共享同一套底层数据**，是草稿箱的 B 端视图。来自批量任务、手动创作、主产品同步的内容均沉淀于此，支持长期回溯。

- 主产品草稿箱中的内容在内容库中可见，反之亦然
- 矩阵模块批量产出内容在主产品草稿箱中折叠展示，不与单篇内容混排
- 内容库承担**沉淀、回溯、资产管理**职责；所有状态型操作（审核、发布、重试）均在批量任务模块中完成，内容库不提供状态操作入口

### 4.2 页面结构：两个 Tab

| Tab | 内容范围 | 说明 |
|-----|----------|------|
| **单篇资产** | 手动创作 + 主产品同步内容 | 扁平卡片列表 |
| **矩阵批次** | 批量任务生产的全部内容 | 按批次任务分组，默认折叠 |

> 已发布与未发布内容均在两个 Tab 中完整呈现，内容库不做状态过滤。

### 4.3 筛选与检索

**单篇资产 Tab**
- 关键词搜索（标题）
- 平台筛选
- 排序（更新时间 / 创建时间）

**矩阵批次 Tab**（在以上基础上额外提供）
- 批次任务搜索（可搜索输入框，支持按任务名关键字筛选，可扩展至 100+ 批次）

### 4.4 矩阵批次 Tab：折叠分组交互

- 默认状态：所有批次折叠为一行，展示批次名、Skill、创建日期、前 3 张封面缩略图、篇数
- 点击行头展开该批次的内容卡片网格
- 可同时展开多个批次
- 批次搜索框选中某批次后自动展开对应批次

### 4.5 内容卡片字段

| 字段 | 说明 |
|------|------|
| 封面图 | 无封面时显示类型占位图标 |
| 内容类型 | 轻图文 / 千字长文，叠加在封面右上角 |
| 标题 | 最多两行截断 |
| 平台 + 账号 | 目标发布平台及账号名 |
| 发布归因 | 已发布：显示发布日期；未发布：显示「未发布 · 已沉淀」 |
| 更新时间 | 卡片右下角 |

> 卡片上**不展示状态标签**（草稿/待审核/就绪/已发布等），状态操作均在批量任务模块处理。

### 4.6 内容详情抽屉

点击内容卡片，右侧滑出详情抽屉，包含：
- 封面图
- 标题、来源属性（平台 / 类型 / 来源）
- 完整正文
- 发布归因信息（目标账号、发布平台、发布时间、最近更新时间）
- 上下篇翻页（键盘 ← → 支持）
- 底部操作：导出（文案 + 图片 ZIP）

### 4.7 内容来源

| 来源 | Tab 归属 |
|------|----------|
| 批量任务生产 | 矩阵批次 Tab |
| 手动创作 | 单篇资产 Tab |
| 主产品同步（来自绘文） | 单篇资产 Tab |

## 五、Skill 体系

### 三层结构

| 层级 | 说明 | 创建者 |
|------|------|--------|
| 原子 Tool | 单一能力（改写、提标题、生图等） | 官方 |
| 单一 Skill | 完整工作流 | 官方 |
| 组合工作流 | 多 Skill 串联，可含发布能力 | 官方 + 用户（二期）|

### Skill 关键属性

- 名称、描述、适用平台
- 输入参数定义（决定参数表格列结构）
- **是否具备发布能力**（`has_publish_capability`）：
  - `false`：新建批量任务 Step 3 为纯内容参数表格；Skill 试用为单阶段（生成后展示结果，提供「存入内容库」与「导出」操作）。
  - `true`：新建批量任务流程为生成+发布型，Step 3 含「发布账号」列与行级发布配置；Skill 试用为两阶段（Phase 1 生成预览 → Phase 2 确认发布）。发布配置跟随账号，不设全局默认。
